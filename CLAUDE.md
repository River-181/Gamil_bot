# CLAUDE.md

## 1) Workspace 운영 원칙
- 이 워크스페이스는 `gmail-agent-sys` 오케스트레이션의 기준 문서 집합이다.
- 현재 단계는 **학습/준비(설계·구현 미실행)**이며, 외부 자동화(OpenClaw, Zapier, 웹훅 기반 자동전송)는 아직 연결하지 않는다.
- Gmail 계정은 사용자 단일 계정 기준(`chany010713@gmail.com`) 정책을 기본으로 하며, 다중 계정은 2차 단계에서 확장한다.

## 2) 기본 목표
- 목표: 수동 승인 가능한 구조에서 자동화로의 전환이 쉬운 Gmail 컨설팅 운영 체계를 설계한다.
- 현재 산출물:
  - 라벨/필터 정책의 정합한 JSON 스키마와 규칙 집합
  - MCP/Skill/Hook/Persona 계약 정의
  - 운영 runbook와 리뷰/검증 시나리오
- 구현은 다음 페이즈에서 시작한다.

## 3) 제약
- 실 API 호출은 지금 단계에서 수행하지 않는다.
- 토큰/비밀번호/클라이언트 시크릿 본문 노출 금지.
- 실행 시 토큰은 사용자 홈 영역 로컬 credential store 또는 환경 변수로만 관리한다.
- 외부 자동화 호출은 `plan-only` 또는 문서 템플릿 수준에서만 기술한다.

## 4) 우선순위
1. 보안 및 개인 정보 보호
2. 정책 정합성(라벨/필터 일관성)
3. 드라이런/검증 가능한 운영 절차
4. 협업 에이전트 간 컨텍스트 정합성

## 5) 언어/출력 규칙
- 기본 응답 언어: 한국어
- 정책 관련 출력은 근거 파일 경로를 함께 표기한다.
- 변경 제안은 `요약 -> 차이점 -> 영향도 -> 다음 단계` 순으로 말한다.
