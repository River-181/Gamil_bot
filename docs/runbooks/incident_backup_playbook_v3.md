# Incident & Backup Playbook (v3)

## 목적
실제 운영 중 문제 발생 시 즉시 중단, 영향 축소, 복구, 재발 방지까지 표준 절차로 처리한다.

## 사고 등급
1. P0
- 보안 라벨 누락/삭제, 대규모 오분류, 개인정보 노출 가능성
- 즉시 중단 및 롤백

2. P1
- 우선순위 충돌로 잘못된 라우팅 반복
- 적용 중단 후 수동 복구

3. P2
- 문서/로그 불일치, 경미한 분류 정확도 저하
- 운영 유지 가능, 다음 릴리즈에서 보정

## 즉시 대응 절차
1. apply 중단
2. 최근 실행 배치 식별(시간/대상/규칙)
3. 영향 범위 산정(메시지 수, 라벨 수, 인박스 영향)
4. 복구 모드 전환

## 백업 전략
1. 정책 백업
- 변경 전마다 `labels.v3.json`, `filters.v3.json` 스냅샷 보관

2. 실행 로그 백업
- 적용 대상 메시지 ID
- 변경 전/후 라벨 상태
- 실패 코드 및 재시도 내역

3. 보관 원칙
- 최소 최근 5회 파일럿/적용 기록 유지
- 민감정보 마스킹 상태로 저장

## 롤백 절차
1. 최근 승인된 정책 버전으로 복원
2. 문제 배치에 적용된 변경 라벨 되돌림
3. 보안 라벨 계열 우선 복구
4. 롤백 완료 후 `plan-only`, `dry-run` 재검증

## 커뮤니케이션 규칙
1. 사고 발생 시 즉시 상태 공유
2. 임시 우회 정책 적용 여부 명시
3. 재실행 가능 조건 충족 전 apply 재개 금지

## 재발 방지
1. 원인 분석(규칙/우선순위/데이터/운영절차)
2. 정책 문서와 체크리스트 업데이트
3. 다음 릴리즈에 회귀 방지 항목 추가
