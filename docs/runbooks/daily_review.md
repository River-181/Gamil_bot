# 일일 리뷰 Runbook

## 목적
매일 인박스를 정비하고, 인공지능 자동분류만으로는 처리되지 않은 수동 판단 항목을 정리한다.

## 준비
- 실행 시간: 매일 동일 시간(권장 오전)
- 필요 파일: `config/labels.v3.json`, `config/filters.v3.json`

## 절차
1. Gmail 인박스 스냅샷 생성
2. 다음 라벨 존재 비율 점검
   - `@CNU/학생`
   - `@SYS/Security`
   - `@AUTO/Newsletter`
   - `@AUTO/Promo`
3. 신규 수신 메일에서 사람 메일 vs 자동 메일 비율 계산(수동 확인)
4. `@GTD` 라벨 재분류
   - `Action`: 오늘/이번 주 처리 대상
   - `Waiting`: 답변 대기 메일
   - `Reference`: 보존형 문서
   - `Read`: 종결형 읽기만 처리
5. 예외/드리프트 표기
   - 인박스가 과다한 라벨(Noise) 항목
   - 충돌로 보이는 필터 결과(동일 대상 다중 라벨)
6. 다음 실행 계획 기록
   - 변경 요청이 있으면 Phase 2 검토 리스트에 반영

## 산출물
- `오늘 처리 대상 수`, `현재 대기 항목 수`, `예외 건수`
- 이상 징후가 있으면 주석(필요 시 필터 우선순위 제안)
