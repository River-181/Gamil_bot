# Phase 7 파일럿 Apply Runbook (v3)

## 목적
실제 Gmail mutate 이전에, 승인 기반으로 소량 범위에서 정책 적용 동작을 검증한다.

## 파일럿 범위(기본값)
- 대상 메일: 최근 24시간 수신 메일 중 최대 20건
- 대상 규칙: 보안/학교/CNU 핵심 규칙 제외 일반 분류 규칙부터 시작
- 적용 방식: 1회 실행 후 결과 리뷰, 반복 자동화 금지

## 사전 조건
1. `tests/plans/phase5_final_verdict_v3.md`가 `PASS`
2. `docs/runbooks/apply_approval_runbook_v3.md` 승인 게이트 통과
3. OAuth 연결 상태 정상 (`connect-check` 통과)
4. 실행 직전 스냅샷 백업 완료

## 백업(필수)
1. 정책 백업
- `config/labels.v3.json`
- `config/filters.v3.json`

2. 실행 컨텍스트 백업
- 적용 대상 메시지 ID 목록
- 적용 예정 라벨/액션 diff
- 실행 시각/실행자/승인자

## 승인 절차
1. 파일럿 범위 명시(건수, 시간창, 적용 규칙)
2. 실패 중단 조건 명시(오분류, 보안 라벨 누락, 비정상 skip)
3. 롤백 조건 명시(오류율 임계치 초과 등)
4. 사용자 최종 승인 문구 확인 후 실행

## 실행 후 점검
1. 적용 성공/실패 수 집계
2. 예외 메시지 추출 및 원인 분류
3. 라벨 충돌/누락 여부 확인
4. 변경 사항과 영향도 요약 기록

## 중단 기준
- 보안 관련 분류 오동작 1건 이상
- 예상치 못한 인박스 누락 징후 발견
- API 오류율이 연속 임계치 초과

## 종료 조건
- 파일럿 결과가 승인 기준 충족
- 다음 단계(확대 적용/보류/정책 수정) 결정 기록
