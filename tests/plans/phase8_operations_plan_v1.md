# Phase 8 실행 계획 v1 (운영 안정화)

## 목표
- Phase 7-b 파일럿 성공 결과를 운영 루틴으로 전환한다.
- 오분류/누락/보안 사고를 조기 감지할 수 있는 모니터링과 백업 체계를 정착한다.
- 확장 적용 전 회귀 방지 게이트를 고정한다.

## 범위
1. 운영 모니터링
- 일일 체크: 적용 건수, 실패율, 보호 스킵 건수, 인박스 누락 징후
- 주간 체크: 신규 발신자 패턴, 미매칭 증가, 규칙 충돌 재발 여부

2. 백업/복구
- 실행 전 정책 스냅샷 자동 기록
- 최근 실행 로그 5회 보관
- P0/P1 사고 시 롤백 실행 절차 표준화

3. 품질 회귀 방지
- apply 전 `plan-only`, `dry-run` 필수 통과
- 동일 우선순위 충돌 재검증
- critical 규칙 보호 스킵 정책 유지

## 실행 순서
1. 템플릿 3종 작성
2. 최근 실행 로그를 기준으로 주간 리포트 샘플 1회 작성
3. 회귀 게이트 문서에서 P0/P1 차단 조건 재확인
4. 다음 apply 배치 확대 여부 결정(10건 -> 20건 or 유지)

## 게이트
- P0: 보안 라벨 오작동/중요 메일 누락 징후 -> 즉시 중단
- P1: 실패율 임계치 초과/충돌 재발 -> 배치 규모 유지 또는 축소
- P2: 문서/리포트 누락 -> 보완 후 재판정

## 완료 기준
- 운영 템플릿 3종 준비 완료
- 회귀 게이트 문서화 완료
- 확대 적용 여부가 문서로 결정됨
